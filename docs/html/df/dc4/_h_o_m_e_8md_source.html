14 This is a template for new projects, gives a good CMake base and a few dependencies you most likely want in your project. It also set ups some basic CI builds.
15 It uses "modern" CMake, ie 3.x paradigms, and should be a good starting point for both people willing to learn it and those that want to update/upgrade their CMakeLists.txt !
16 If you disagree with some pieces of advice given here, please discuss it with me by opening a Github Issue ! Enhancements are always welcome.

18 ## Usage
19 
20 If you want to bootstrap a new project you only need to :
21 
22 - If you don't already have your git repository setup
23  - Simply copy/paste the folder (without the .git folder) and run the createBoilerPlate.sh file. This will create an initial git commit and add the _required_ submodules.
24 - Hack CMakeLists.txt and CTestConfig.cmake to change the project name
25 - Ready to go !

40 ## Some features/notes :
41 
42 - Scripts lying in the cmake/ folder can be copy/pasted for use in any CMake project
43 - Uses c++14
44 - CopyDllsForDebug.cmake script : A small wrapper around fixup_bundle to copy DLLs to the output directory on windows
45 - LTO.cmake script : Easier link time optimization configuration (should work on all CMake 3.x versions) as it used to be painful to setup.
46 - Warnings.cmake script : A wrapper around common warning settings
47 - Basic unit-testing using [doctest](https://github.com/onqtam/doctest)
48 - Coverage.cmake : Test coverage script to add a 'Coverage' build type to CMake
49 - Generating the documentation through TravisCI requires that you setup a Github Token, see <https://docs.travis-ci.com/user/deployment/pages/> .
50 - The lgtm.com website already knows about cmake and can build most of the projects without any special configuration. A sample configuration is in this project to show how to tag files and disable unneeded warnings. The external folder is automatically recognized and files are tagged as library.

60 **Q**: Why can't I link some new libraries I put inside the external folder ?
61 
62 **A**: By default targets are not at the GLOBAL scope, which means your CMakelists.txt might not see it.
63 In this case you can either add an alias/imported library or use find_package/library as you would if the library was not in your buildtree.
64 
65 ___
66 
67 **Q**: Should I always put my dependencies in the folder external
68 
69 **A**: Absolutely not ! It is a great place for small libraries, but you probably don't want to have to rebuild big libs every time.
70 For those, you can use a package manager such as [Hunter](https://github.com/ruslo/hunter) or simply rely on find_package/library.
